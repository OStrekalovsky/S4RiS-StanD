    • Presenter Mod

      Modification of S4RiS StanD for defrosting with presenter.
      Navigation:
      • Next Step - button "Next" on presenter or Pade Down on keyboard
      • Fast Next Step - button "Back" on presenter or Pade Up on keyboard
      • Back step - temporary only by keyboard button "B".

    JSON log format

    Specification

    {
   "contestName": <Contest caption>,
   "problems": <Array of strings of problem names>,
   "contestants": <Array of strings of contestant's names>,
   "freezeTime": <Freeze time of contest standings>,
   "runs":[
      {
         "contestant":<Contestant name>,
         "problem":<Name of problem>,
         "time":<Minute of submission>,
         "success":<Solution was accepted or not>
      },
   ]
}

    Notes

    • Due possilibity to have many submissions in one minute it's recommend to sort your submissions as they arrived for checking.
    • Pay attention to rounding rules of submission's second to minutes.
    • It use standard algorithm of sorting contestant's results of ACM ICPC contest:
      • Sort by decreasing number of solved problems.
      • Sort by increasing penalty.
      • Sort by increasing last successful submission.
      • Contestants with the same success recieved the same places.

    Example

    {
   "contestName": "Interuniversity programming contest",
   "problemLetters": ["A","B","C","D","E","F","G","H"],
   "contestants":
    [
        "Strekalovsky Oleg. Vologda SPU", 
        "Fyodor Menshikov. Vologda SPU",
    ],
   "freezeTimeMinutesFromStart": 1,
   "runs":[
      {
         "contestant":"Strekalovsky Oleg. Vologda SPU",
         "problemLetter":"A",
         "timeMinutesFromStart":10,
         "success":false
      },
      {
         "contestant":"Fyodor Menshikov. Vologda SPU",
         "problemLetter":"A",
         "timeMinutesFromStart":15,
         "success":true
      },
   ]
}
